# Input

> [!abstract] C++ 输入笔记
> 关于 `scanf` 与数组指针的等价写法，理解数组名退化为指针的机制。

## 数组名与 `scanf`

```cpp
int d[100];

// 以下几种写法等价
scanf("%d", d);        // 正确：d 退化为指向 d[0] 的指针
scanf("%d", &d[0]);    // 正确：显式取第一个元素的地址
scanf("%d", d + 0);    // 正确：指针运算
```

> [!warning] 常见错误
> ```cpp
> scanf("%d", &d);  // ❌ 类型是 int (*)[100]，不匹配 %d 的 int* 要求
> ```
> `&d` 的类型是 `int (*)[100]`（指向整个数组的指针），而非 `int*`。虽然编译会通过，但会产生警告，且行为未定义。

## 核心概念

| 表达式 | 类型 | 说明 |
|--------|------|------|
| `d` | `int*` | 数组名退化为指向首元素的指针 |
| `&d[0]` | `int*` | 显式取首元素地址 |
| `d + 0` | `int*` | 指针运算，等价于 `d` |
| `&d` | `int (*)[100]` | 指向整个数组的指针，**不是** `int*` |

> [!tip] 记忆要点
> 数组名在大多数情况下会**退化为指向首元素的指针**。唯一例外是 `sizeof`、`&` 和字符串字面量初始化。
