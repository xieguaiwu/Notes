## 2. 理论基础

### 2.1 乔姆斯基层级

$$
\begin{array}{c}
\text{0型：递归可枚举语言} \leftarrow \text{图灵机} \\
\uparrow \\
\text{1型：上下文有关语言} \leftarrow \text{线性有界自动机} \\
\uparrow \\
\text{2型：上下文无关语言} \leftarrow \text{下推自动机} \\
\uparrow \\
\text{3型：正则语言} \leftarrow \text{有限状态机}
\end{array}
$$

> **IOLC计算题主要涉及**：3型（正则语言）和2型（上下文无关语言）

### 2.2 形式语言定义回顾

**语法四元组**：
$$G = (V_N, V_T, S, P)$$

| 元素 | 含义 | 说明 |
|------|------|------|
| $V_N$ | 非终极符号 | 不能处于生成过程的终点 |
| $V_T$ | 终极符号 | 处于生成过程的终点 |
| $S$ | 起始符号 | 生成过程的起点 |
| $P$ | 重写规则 | 形如 $\phi \to \psi$ |

**语言定义**：
$$L(G) = \{w \mid w \in V_T^* \wedge S \Rightarrow^*_G w\}$$

## 3. 有限状态机（FSM）

### 3.1 定义

$$M = (Q, \Sigma, \delta, q_0, F)$$

| 元素 | 含义 | 语言学对应 |
|------|------|------------|
| $Q$ | 有限状态集合 | 构词阶段/音系环境 |
| $\Sigma$ | 输入字母表 | 音素/语素/词素 |
| $\delta: Q \times \Sigma \to Q$ | 转移函数 | 规则应用 |
| $q_0 \in Q$ | 初始状态 | 起点 |
| $F \subseteq Q$ | 接受状态集合 | 合法终点 |

### 3.2 状态转移图表示

**构词一致性检测示例**：

```
检测CVCV结构 + 后缀的词

        C        V        C        V        S
     ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐
     │     │ │     │ │     │ │     │ │     │
   (q0)───→(q1)───→(q2)───→(q3)───→(q4)───→(q5)[接受]
     │                ↑
     └────────────────┘
         (允许重复CVCV)
```

**状态转移表**：

| 当前状态 | 输入 | 下一状态 |
|----------|------|----------|
| q0 | C | q1 |
| q1 | V | q2 |
| q2 | C | q3 |
| q3 | V | q4 |
| q4 | C | q3 |
| q4 | S | q5 |
| q5 | S | q5 |

### 3.3 FSM在语言学中的应用

| 应用场景 | 描述 | 示例 |
|----------|------|------|
| **构词一致性** | 检测词形是否符合构词规则 | 闪含语系辅音根 + 元音模式 |
| **音系规则** | 形式化音变过程 | k → g / V_ |
| **形态分析** | 识别词的内部结构 | 词根 + 词缀的组合模式 |
| **音节结构** | 验证音节合法性 | CV, CVC, V 等模式 |

### 3.4 音系规则的FSM实现

**规则**：$k \to g \ / \ V\_$ （元音后k浊化）

## 4. 上下文无关文法（CFG）

### 4.1 与FSM的对比

| 特性 | FSM（正则） | CFG（上下文无关） |
|------|-------------|-------------------|
| **记忆能力** | 无 | 有（栈） |
| **嵌套结构** | 不能处理 | 可以处理 |
| **表达能力** | 较弱 | 较强 |
| **应用** | 构词、音系 | 句法、嵌套结构 |

### 4.2 句法分析应用

**句法规则示例**：
```
S → NP VP
NP → Det N | Det N PP | NP Conj NP
VP → V | V NP | V NP PP | VP Conj VP
PP → P NP
```

**推导树**：

$$
\begin{array}{c}
\text{S} \\
/|\backslash \\
\text{NP} \quad \text{VP} \\
/|\backslash \quad /|\backslash \\
\text{Det N} \quad \text{V NP} \\
| | | /|\backslash \\
\text{the cat sat PP} \\
/|\backslash \\
\text{P NP} \\
/|\backslash \\
\text{Det N} \\
| | \\
\text{the mat}
\end{array}
$$

## 6. AI语言逻辑题（新题型）

### 6.1 可能的考察方向

| 方向 | 描述 | 示例 |
|------|------|------|
| **规则逆向工程** | 从AI输出推断规则 | 给出AI生成的词形变化，推断输入规则 |
| **一致性检测** | 判断AI输出是否符合语言学规律 | 检测AI生成的词形是否遵循元音和谐 |
| **错误诊断** | 找出AI输出的错误并修正 | 识别AI翻译中的格标记错误 |
| **规则补全** | 填补缺失的规则 | 给出部分规则和样例，补全完整规则系统 |

### 6.2 解题策略

```
AI输出 → 分解成分 → 检查内部一致性 → 对比已知规则 → 发现偏差 → 修正规则
```

### 6.3 示例：辅音根语言分析

**给定AI生成的词形变化**：

| 词形 | 含义 |
|------|------|
| kataba | 他写了 |
| kutiba | 被写了 |
| kaatiba | 作家 |
| maktaba | 图书馆 |

**解题步骤**：

**步骤1**：识别辅音根
- k, t, b 出现在所有词中 → 辅音根 k-t-b

**步骤2**：提取元音模式

| 元音模式 | 派生类型 |
|----------|----------|
| a-a-a | 主动过去时 |
| u-i-a | 被动过去时 |
| aa-i-a | 施事名词 |
| _a_a + ma- | 处所名词 |

**步骤3**：形式化规则
$$\text{Word} = \text{Root}(k, t, b) + \text{Pattern}(V_1, V_2, V_3) + \text{Affix}$$

## 7. 解题模板

### 7.1 构词一致性检测题

```
步骤1：分词
  - 识别词根（Root）
  - 识别词缀（前缀P、后缀S、中缀I）

步骤2：标注
  - 标记每个成分的类型
  - 例：R-CVCV, P-mu-, S-un

步骤3：构建FSM
  - 确定状态Q
  - 确定字母表Σ
  - 确定转移函数δ
  - 确定初始状态q0和接受状态F

步骤4：验证
  - 检查新词是否能被FSM接受

步骤5：输出规则
  规则：[词根类型] + [词缀组合模式]
  约束：[附加条件]
```

### 7.2 形式文法推导题

```
步骤1：确定语法四元组
  VN = {非终极符号集合}
  VT = {终极符号集合}
  S = 起始符号
  P = {重写规则集合}

步骤2：写出推导过程
  S ⇒ α1 ⇒ α2 ⇒ ... ⇒ w

步骤3：画出推导树（如需要）

步骤4：回答问题
  - 生成：写出能生成的所有句子
  - 判断：给定句子是否属于L(G)
  - 计数：统计满足条件的句子数量
```

### 7.3 规则发现题

```
步骤1：收集数据
  - 列出所有给定的对应关系

步骤2：对齐分析
  - 找出变化部分和不变部分

步骤3：提取变量
  - 变量 = 变化部分
  - 常量 = 不变部分

步骤4：归纳规则
  规则形式：如果条件X，则变化Y

步骤5：验证规则
  - 检查所有数据是否满足规则
  - 寻找可能的反例
```

## 8. 形式化表达规范

### 8.1 符号规范

| 符号 | 含义 | 正确示例 | 错误示例 |
|------|------|----------|----------|
| $\to$ | 定义/重写 | $S \to NP\ VP$ | S→NP VP |
| $\Rightarrow$ | 一步推导 | $S \Rightarrow NP\ VP$ | S变成NP VP |
| $\Rightarrow^*$ | 多步推导 | $S \Rightarrow^* w$ | S最终变成w |
| $\in$ | 属于 | $a \in V_T$ | a在VT里 |
| $\mid$ | 或 | $NP \to Det\ N \mid NP\ PP$ | NP是Det N或NP PP |

### 8.2 常见错误避免

| 错误类型 | 示例 | 修正 |
|----------|------|------|
| 符号混淆 | 混用$\to$和$\Rightarrow$ | $\to$用于规则定义，$\Rightarrow$用于推导过程 |
| 状态遗漏 | FSM忘记终止状态 | 检查所有路径是否可达接受状态 |
| 规则冲突 | 两条规则对同一输入给出不同输出 | 添加优先级或条件约束 |
| 过度泛化 | 从少量数据得出过于宽泛的规则 | 检查反例，限制条件 |
| 集合表示 | $V = a, b, c$ | $V = \{a, b, c\}$ |
